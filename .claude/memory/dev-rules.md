# 开发规则

## Git 规则
- **禁止回滚 git**：除非明确指示，否则禁止 `git reset --hard`、`git revert`、`git checkout --`
- **渐进式迭代**：所有变更必须增量推进，每步可追溯
- **提交前审查**：`git status` 检查待提交文件

## 代码质量规则
- **禁止兜底冗余代码**：错误应当及时暴露而非被静默吞掉
- **禁止简化业务需求**：复杂场景必须完整实现
- **能力不够就扩展，不绕过**：Widget 不满足就创建新 Widget

## 架构规则
- 全局状态统一使用 Pinia Store
- 公共逻辑统一使用组合式 API（useXXX）
- API 接口必须聚合到 `src/api/`，禁止组件直接 fetch
- UI 组件只做渲染，不写复杂业务逻辑

## Widget 开发规则
- 每个 Widget 独立目录：`src/widgets/[name]/`
- 必须包含：`index.vue` + `config.ts`
- 容器禁止互相嵌套
- 样式 100% Schema 驱动，删除所有硬编码样式
- 每个组件必须有对应属性配置面板

## 验证清单
每次变更后必须执行：
1. `pnpm build:check` — 类型检查 + 构建
2. `pnpm test` — 运行测试用例
3. 检查属性面板配置项能正常生效
4. 检查样式由 Schema 驱动，无硬编码
5. 检查容器嵌套规则未被违反
